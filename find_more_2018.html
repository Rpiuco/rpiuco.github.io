piRNAdb Supplementary Methods

Data Sources

We colleted RNA sequences from the NCBI Nucleotide database for the H. sapiens, M. musculus, R. norvegicus, C. griseus and C. elegans, but only those sequences classified as as a piRNA. Additionally, for humans we got sequences from supplementary results of 5 published papers. Finally, the D. melanogaster sequences were download from ENA database, following the same criteria used for other species.

We also look for piRNA data on RNACentral and RNAdb databases. We did that to compare the sequence bases and accession codes from each piRNA stored on different databases, and after that we had a listo f piRNA sequences and their access codes on each database that we used to collect these RNAs. This results was used on the preparation of the cross codes feature, more details below.

Reference genome sequences of Homo sapiens (hg19, hg38), Mus musculus (mm10), Rattus norvegicus (rn6), Drosophila melanogaster (dm6) and Caenorhabditis elegans (ce11) were obtained from UCSC genome Browser (http://hgdownload.cse.ucsc.edu). For the C. griseus specie we do not found a oficial genome file, so at this step we continued only with the other 5 species. Coordinates from the organism genes were downloaded from EMSEMBL database (XXX). We also collected data from gene ontology terms for each organism genes from GO database (XXX), this information is available at the ontology term feature on the piRNA specific page.

Procedures

On each species, we merged the datasets and removed the sequences found duplicated, after thsi filtering step, these sequences were aligned to the genome using the software BWA(XXX). At this step, we do no allow mismatches nor gaps. piRNA sequences that do not had any alignment to its genome were removed from the database. All the alignment positions were stored on database and displayed at the piRNA page.

Then we used this alignment data to find any gene or transposable element that is located at the same position and strand of alignment. This information is available at the alignment tab on the piRNA page.

We then associate the gene ontology terms to the genes that were found at the last step. All these information are stored and provided to the users at the piRNA page.

piRNA expression

For humans, we colleted data from microRNA sequencing (miRNA-seq) from 4 tissues (BRCA, COAD, PRAD and PAAD) available at The Cancer Genome Atlas (TCGA, https://portal.gdc.cancer.gov/). At this time, we separated the samples between control and tumor and begin the analysis. First step was to get all sequences from a .bam file do a fastq file. Then we aligned the short sequences to the human genome hg38 and use these aligned files as a parameter to the software intersectBed from Bedtools(XXX). We also provided a file containing and piRNAs of piRNAdb and the software associated each read from the aligned sequences to the piRNAdb sequences. The final result was the raw count for every piRNA found at the control and tumor samples.

We then eu this count file to use into the edgeR package (XXX) from R software to get the TMM values. This TMM normalization method is already cited to be the best to use on short RNA sequeneces, like piRNAs and miRNAs, it is done using the size of library and the raw counts to make the samples with more coverage comparable to those with less coverage.

Data from raw Counts and TMM values are available at the piRNA page on expression section. To change from one type to another just click at the buttom for the name.

We done this expression evaluation step to other 5 species that we have at our database, the methodology was almost the same. The major changes are the version of the genome that we used to align the sequences, each organismo have its own genome version, and the source of sequeneces. CrossCodes

We got all piRNA sequences and access codes from other databases, like NCBI, RNACentral and piRNABank and merged them based on the sequences bases. After this step, we provide this information on two section, first at the piRNA lais abox on the piRNA information and the second at the cross codes section. THis feature provides the user a form top lace all their piRNA accession codes from other databases and after some data process the result a listo f piRNA code that we use on piRNAdb.
